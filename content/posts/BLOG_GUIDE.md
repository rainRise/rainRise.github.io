=================================================================
博客写作与发布操作指南
用于：Mizuki 博客项目
创建时间：2026-03-27
=================================================================

一、博客文章存放位置
=================================================================
文章文件应放在：src/content/posts/ 目录

文件格式：.md (Markdown)

文章头部必须包含以下元数据（Frontmatter）：

---
title: 文章标题
published: 2026-03-27
description: 文章描述
image: '/assets/封面图片路径.jpg'
tags: [标签1, 标签2]
category: 分类名称
draft: false
lang: 'zh-CN'
---

=================================================================
二、封面图片存放位置
=================================================================
封面图片应放在：public/assets/ 目录下的任意位置

推荐存放目录：
- public/assets/ (根目录)
- public/assets/desktop-banner/ (桌面横幅目录)
- public/assets/mobile-banner/ (移动端横幅目录)

图片格式支持：jpg, png, webp, gif

在文章中引用方式：
- 相对路径（推荐）：image: '/assets/xxx.jpg'
- 绝对路径：image: '/assets/desktop-banner/1.jpg'

=================================================================
五、注意事项
=================================================================
1. 图片必须放在 public/ 目录下，文章中用 /assets/... 路径引用
2. 文章中的 image 路径是相对于 public 目录的，即 /assets/xxx.jpg
3. published 日期格式必须为 YYYY-MM-DD
4. draft: true 时文章不会发布，false 时才会发布
5. 推送后GitHub会自动构建部署，无需手动构建

=================================================================
六、常见问题
=================================================================
Q: 文章写完后没显示？
A: 检查 draft: false 是否设置，published 日期是否正确

Q: 封面图片不显示？
A: 检查图片是否在 public/ 目录，路径是否正确

Q: 部署失败？
A: 检查 .md 语法是否有错误，查看 GitHub Actions 日志

七、markdown基本语法
=================================================================
1. 标题
```
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```
2. 文字样式
```
**加粗**
*斜体*
***粗斜体***
~~删除线~~
<u>下划线</u>
```
3. 列表
```
无序列表
- 列表项1
- 列表项2
  - 子列表
有序列表
1. 第一项
2. 第二项
3. 第三项
```
4. 引用
```
> 这是一段引用
>> 嵌套引用
```
5. 表格
```
| 姓名 | 年龄 |
| ---- | ---- |
| 张三 | 20   |
```
6. 任务列表
```
- [x] 已完成
- [ ] 未完成
```
7. 软换行
```
行尾加两个空格再回车 = 软换行
```
